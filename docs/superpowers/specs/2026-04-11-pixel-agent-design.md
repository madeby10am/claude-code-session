# Pixel Agent — VS Code Extension Design

**Date:** 2026-04-11
**Status:** Approved

---

## Overview

A VS Code extension that opens a square webview panel in a split editor column showing a pixel art developer character ("Classic Dev") sitting at a desk. The character has six animation states that change based on VS Code editor activity and Claude Code agent activity.

---

## Character

- **Style:** Modern indie pixel art, 64×64px sprites
- **Look:** Classic Dev — dark gray hoodie, messy brown hair, glasses, light skin
- **Scene:** Character sits at a desk with a monitor; dark room background (`#0d0d1a`)

## Sprite Sheets

Each animation state has its own horizontal PNG sprite strip generated by `scripts/generate-sprites.ts` using `node-canvas`. No external art tool required. Sheets are committed to `assets/sprites/`.

| State | Frames | Playback |
|---|---|---|
| `typing` | 4 | Loop |
| `drinking_coffee` | 6 | Play-once, hold last frame until transition |
| `leaning_back` | 3 | Loop |
| `stretching` | 6 | Play-once, then snap to `typing` |
| `walking` | 6 | Loop |
| `sleeping` | 2 | Loop (slow ZZZ pulse) |

CSS animation uses `steps(N)` + `background-position` to advance frames. The webview receives a `{ type: 'setState', state: string }` message and swaps a `data-state` attribute on the character element; CSS handles frame stepping.

## State Machine

Managed in the extension host (`stateManager.ts`). Exactly one state is active at a time.

### Transitions

```
[any state]           + keypress               → typing
typing                + 30s idle               → drinking_coffee
drinking_coffee       + animation done         → leaning_back  (if still idle)
leaning_back          + 5min idle              → walking
walking               + 10min idle             → sleeping
typing                + 15min continuous       → stretching → typing
[claude:active event] + user not typing        → leaning_back  (override)
```

### Rules

- Any keypress from any non-`typing` state immediately snaps back to `typing`
- `stretching` is a one-shot state: plays once then returns to `typing`
- `drinking_coffee` plays its full animation before transitioning to `leaning_back`
- Claude activity overrides idle states but never interrupts `typing` or `stretching`

## Activity Detection

### VS Code (`activityMonitor.ts`)

- `vscode.workspace.onDidChangeTextDocument` → keypress detected, resets idle timer
- Idle timer chain: 30s → `drinking_coffee`, 2min → `leaning_back`, 5min → `walking`, 10min → `sleeping`
- Continuous typing tracked with a `setInterval` counter; counter resets on any idle period
- At 15min of continuous typing, emits `typing:extended` event → triggers `stretching`

### Claude Code (`claudeWatcher.ts`)

- `fs.watch` on `~/.config/claude/logs/`
- Any file modification within that directory → emit `claude:active`
- Debounced 3s: no new writes for 3s → emit `claude:idle`
- `claude:active` triggers `leaning_back` only when user is not in `typing` or `stretching` state

## Webview Panel (`panel.ts`)

- Opened as a **pinned split editor column** (ViewColumn.Two) on extension activation
- Fixed **200×200px** viewport — square, no scrollbars
- Background: dark room scene (`#0d0d1a`) with subtle CSS desk elements
- Character centered horizontally, anchored to bottom of viewport
- `retainContextWhenHidden: true` — panel state survives hide/show
- If the user closes the panel, `extension.ts` reopens it automatically via `onDidDispose`

---

## Extension Activation

- Activates on VS Code startup (`"activationEvents": ["*"]`)
- `extension.ts` instantiates `StateManager`, `ActivityMonitor`, `ClaudeWatcher`, and `Panel` in order
- `ActivityMonitor` and `ClaudeWatcher` emit events that `StateManager` consumes
- `StateManager` calls `Panel.setState(state)` on each transition
- All resources disposed cleanly in `deactivate()`

---

## Out of Scope

- User-configurable character colors or appearance
- Multiple characters
- Sound effects
- Network-based Claude activity detection (file watching only)
